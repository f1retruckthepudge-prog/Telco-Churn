# 📊 Прогнозирование оттока клиентов телеком-компании
### *Telco Customer Churn Analysis*

**Цель:** Найти ключевые драйверы оттока, построить прогнозную модель и предложить меры удержания.

**Инструменты:** SQL (PostgreSQL), Python (pandas, matplotlib, seaborn, scikit-learn), Jupyter Notebook.

---

## 1. Данные
- Источник: IBM Telco Customer Churn
- 7 043 клиента, целевая переменная `Churn` (Yes/No)

---

## 2. Общая картина
| Показатель | Значение |
|------------|----------|
| Отток | 26.5% (1 869) |
| Средний срок жизни | 32.4 мес. |
| Средний ежемесячный платёж | $64.8 |

---

## 3. Ключевые инсайты

### 🔴 Самый опасный сегмент
**Помесячный контракт + Fiber optic → отток 54.6%**  
(на двухлетнем контракте с тем же интернетом — всего 7.2%)

![Отток по контрактам и интернету](images/contract_internet_heatmap.png)

### 🛡 Отсутствие безопасности удваивает риск
Для клиентов Fiber optic:
- Без OnlineSecurity — **49.4%** оттока
- С OnlineSecurity — **21.8%**
- Без TechSupport — **49.4%**
- С TechSupport — **22.6%**

![Влияние сервисов](images/services_churn.png)

### 💸 Дорогие клиенты уходят быстрее
Среди помесячных контрактов:
- Оставшиеся: tenure 17.6 мес., чек $61.3
- Ушедшие: tenure 8.5 мес., чек $72.9

![Срок и чек](images/tenure_charges_boxplot.png)

### 🧾 Ручная оплата = маркер оттока
- Electronic check → отток **45.3%**
- Автоплатежи (bank/credit card) → 15-17%

![Оплата и отток](images/payment_churn.png)

## 4. Предсказательная модель (Logistic Regression)

**Подготовка:** преобразованы категориальные признаки, удалены пропуски, масштабирование не потребовалось (для линейной модели).

**Метрики на тестовой выборке (30%):**

| Метрика | Значение |
|---------|----------|
| Accuracy | 0.80 |
| Precision (Yes) | 0.65 |
| Recall (Yes) | 0.56 |
| F1-score (Yes) | 0.60 |
| ROC-AUC | 0.83 |

*Модель лучше случайного угадывания, хорошо выделяет уходящих (AUC 0.83), но можно улучшить (см. SHAP-анализ).*

### Важность признаков (SHAP)
![SHAP importance](images/shap_global_importance.png)
*(замени на своё изображение после выполнения SHAP)*

**Выводы по модели:**
- Наибольший вклад вносят: `Contract`, `InternetService`, `PaymentMethod`, `Tenure`.
- Подтверждается, что контракт и доп. сервисы критичны.

## 5. Рекомендации и экономический эффект

| Мера | Ожидаемый эффект | Доп. сохранённые клиенты/год |
|------|------------------|---------------------------|
| «Безопасный старт» – бесплатный OnlineSecurity на 3 мес. новым клиентам помесячного Fiber optic | Снижение оттока с 54.6% до ~35% | ~400 |
| Стимулирование годовых контрактов (скидка 10% на 3-й месяц) | Снижение до ~11% | ~190 |
| Пакет лояльности текущим клиентам без безопасности (месяц бесплатно) | ~25 п.п. | ~250 |
| Поощрение автоплатежа (скидка $2/мес.) | ~25 п.п. | ~500+ |

**Расчёт:** средний LTV клиента ≈ $2 100 (32.4 × 64.8). Доп. сохранённые ~840 клиентов × $2 100 = **$1.76M ежегодной дополнительной выручки** (оценка).

## 7. Как запустить проект
1. Установите зависимости: `pip install -r requirements.txt`
2. Поместите `telco.csv` в папку `data/` (файл можно скачать по ссылке в источниках).
3. Запустите SQL-скрипты из `sql/` в PostgreSQL.
4. Откройте `churn_model.ipynb` и выполните ячейки.
